3.6.1 Imaging Status State Machine
Imaging items cycle through confirmed multi-state statuses. UPDATED in v1.1:
	•	Ordered — order placed, timestamp shown
	•	Pending tech — technician not yet started
	•	In scanner — patient actively being scanned
	•	Rad reading — image acquired, awaiting radiologist read
	•	ALERT: >30m — read threshold exceeded, escalation triggered (shown in red)

3.6.2 Lab Alert Thresholds
Labs show inline alert rules directly in the pending rail:
	•	Trop #2 — 3hr delta draw due [time] · Alert on critical value
	•	Lactate — Initial: [value] with delta indicator · CRITICAL — sepsis bundle! (red, lightning bolt icon)
	•	D-Dimer — Pending · Alert if >500

6. AI System Design
6.1 LAW Panel Architecture
The LAW panel is the primary AI interface. It is a streaming text workspace — not a card-based UI. Key implementation requirements:
	•	Render output in JetBrains Mono or similar monospace font, green on dark background
	•	Stream tokens as they arrive — do not wait for full response
	•	Output area is a scrollable div with text-select enabled
	•	Input field at bottom — persistent, always-visible text prompt
	•	Context auto-loaded when AI Assist chip clicked — patient data injected into prompt
	•	Mode badge 'WORKFLOW ONLY · NO AUTO-ORDERS' always visible — cannot be hidden
	•	'LAW' label on left edge of panel — named component for routing and accessibility

6.3 Prompt Safety Architecture
	•	System prompt enforces: WORKFLOW ONLY, NO AUTO-ORDERS, NO DIAGNOSIS, NO PRESCRIBING
	•	All responses parsed against Zod schema — invalid or unsafe responses discarded
	•	Every call logged to law_interactions table — full audit trail
	•	Patient identifiers in prompts use encounter ID only — no MRN, DOB, SSN
	•	Risk summary output uses consistent colour-code tokens mapped client-side — AI outputs patient labels, client applies colour
	•	AVS templates flagged '(review required)' — clinician must approve before use

8. Security & Compliance
8.1 Access Control
	•	Roles: ADMIN · MD (Physician) · PA/NP · RN (Nurse) · CHG (Charge Nurse) · READONLY
	•	Row-Level Security on all Postgres tables — users access only their department's data
	•	JWT-based sessions, 8-hour expiry with refresh
	•	DR. [initials] in header always reflects the current authenticated session
	•	Full audit log: all data access, AI queries, and patient record changes

8.2 AI Safety Controls
	•	System prompt enforces workflow-only mode — cannot be overridden by user input
	•	All AI API calls proxied server-side — API keys never exposed to browser
	•	law_interactions table logs every prompt hash, response, provider, and latency
	•	AVS and handoff outputs marked as drafts requiring clinical review before use
	•	Graceful degradation: if AI API unavailable, system operates in manual mode
